# Traceability Matrix (Core v0 Flow)

This matrix traces core v0 runtime directives across implementation, behavior specs, checks, and runtime evidence.

| Source directive | Runtime enforcement function | BDD scenario ID | Unit/eval check | Log event evidence |
|---|---|---|---|---|
| **Ingest**: every user utterance is ingested as memory input and recorded for auditability before downstream processing. | `append_session_log("user_utterance_ingest", ...)`, `make_utterance_card(...)`, `store_doc(...)`, `generate_reflection_yaml(...)` in `src/testbot/sat_chatbot_memory_v2.py`. | `BDD-MR-01` / `BDD-MR-02` (from `features/memory_recall.feature` scenarios that assume deterministic memory fixtures are loaded before ask/answer). | `behave features/memory_recall.feature` (fixture loading + ask flow contract). | `user_utterance_ingest` row in `logs/session.jsonl`. |
| **Intent**: each utterance is deterministically classified before downstream stages to separate control/meta/memory/knowledge behaviors. | `classify_intent(...)`, `_intent_label(...)`, and `append_session_log("intent_classified", ...)` in `src/testbot/sat_chatbot_memory_v2.py` + enum definitions in `src/testbot/intent_router.py`. | No dedicated BDD scenario yet (covered indirectly by utterance families in memory + answer scenarios). | `pytest tests/test_intent_router.py tests/test_runtime_logging_events.py::test_intent_classified_log_contains_bandit_fields` (routing + logging payload checks). | `intent_classified` row in `logs/session.jsonl` (`intent`, `ambiguity_score`, `user_followup_signal_proxy`). |
| **Rewrite**: user utterance is rewritten into a short retrieval query and the rewrite is captured. | `QUERY_REWRITE_PROMPT` + `llm.invoke(...).content.strip() or utterance`; `append_session_log("query_rewrite_output", ...)` in `src/testbot/sat_chatbot_memory_v2.py`. | `BDD-MR-01` / `BDD-MR-02` (shared ask flow; no rewrite-only scenario yet). | `behave features/memory_recall.feature`; `python scripts/eval_recall.py` (query interpretation alignment pressure via retrieval outcomes). | `query_rewrite_output` row in `logs/session.jsonl`. |
| **Retrieve**: retrieval uses similarity search candidates and confidence gating to control answerability. | `store.similarity_search_with_score(query, k=12)`, `has_sufficient_context_confidence(...)`, `append_session_log("retrieval_candidates", ...)` in `src/testbot/sat_chatbot_memory_v2.py`. | `BDD-MR-01` (sufficient context -> grounded answer) and `BDD-MR-02` (insufficient context -> exact fallback). | `behave features/memory_recall.feature`; `python scripts/eval_recall.py --top-k 4` (`hit_at_k`, rank tracking). | `retrieval_candidates` row in `logs/session.jsonl`. |
| **Rerank**: candidate ordering is time-aware (target-time parse + adaptive sigma + type/time weighting). | `parse_target_time(...)`, `adaptive_sigma_fractional(...)`, `rerank_docs_with_time_and_type(...)`, `append_session_log("time_target_parse", ...)` in `src/testbot/sat_chatbot_memory_v2.py`. | `BDD-MR-01` / `BDD-MR-02` (indirectly covered through end-to-end recall behavior; no rerank-specific scenario yet). | `python scripts/eval_recall.py` (`parse_target_time` adapter + ranking metrics); internal scoring path in `candidate_score(...)`. | `time_target_parse` row in `logs/session.jsonl`. |
| **Answer mode**: answer must be memory-grounded with required citation fields, otherwise exact fallback `I don't know from memory.` and explicit mode logging. | `validate_answer_contract(...)`, `response_contains_claims(...)`, `has_required_memory_citation(...)`, confidence gating + fallback branches, `append_session_log("final_answer_mode", ...)` in `src/testbot/sat_chatbot_memory_v2.py`. | `BDD-MR-01` (cited grounded answer), `BDD-MR-02` (exact fallback), `BDD-AC-01` (uncited factual response rejected). | `behave features/memory_recall.feature features/answer_contract.feature`; contract mirror in `features/steps/answer_contract_steps.py`. | `final_answer_mode` row in `logs/session.jsonl` (`mode`, `context_confident`, `retrieved_docs`). |
| **Fallback policy decision**: selected policy action is logged with optimization-ready features for downstream bandit/offline analysis. | `decide_fallback_action(...)` in `stage_answer(...)`, `_user_followup_signal_proxy(...)`, and `append_session_log("fallback_action_selected", ...)` in `src/testbot/sat_chatbot_memory_v2.py`. | `BDD-MR-02` / `BDD-AC-01` (fallback/clarify behavior exercised via shared answer path). | `pytest tests/test_reflection_policy.py tests/test_runtime_logging_events.py::test_fallback_action_selected_log_contains_bandit_fields`. | `fallback_action_selected` row in `logs/session.jsonl` (`intent`, `ambiguity_score`, `chosen_action`, `user_followup_signal_proxy`). |
| **Provenance summary**: final provenance evidence is emitted with policy/action context to support attribution and optimization instrumentation. | `build_provenance_metadata(...)`, `state.claims/state.provenance_types/state.used_memory_refs/state.used_source_evidence_refs/state.source_evidence_attribution/state.basis_statement`, and `append_session_log("provenance_summary", ...)` in `src/testbot/sat_chatbot_memory_v2.py`. | `BDD-MR-01` (grounded provenance present) and `BDD-MR-02` (fallback provenance mode). | `pytest tests/test_runtime_logging_events.py::test_provenance_summary_log_contains_bandit_and_provenance_fields`; `python scripts/validate_log_schema.py`. | `provenance_summary` row in `logs/session.jsonl` (`intent`, `ambiguity_score`, `chosen_action`, `user_followup_signal_proxy`, `provenance_types`, `used_memory_refs`, `used_source_evidence_refs`, `source_evidence_attribution`). |
| **Analytics replay loop**: per-turn policy telemetry is aggregated into KPI evidence for policy evolution and release drift review. | `aggregate_turn_dataset(...)` + `compute_kpis(...)` in `scripts/aggregate_turn_analytics.py`; optional replay-report checks integrated in `scripts/all_green_gate.py` via KPI guardrail modes. | No dedicated BDD scenario yet (observability/evidence layer on top of existing answer flow scenarios). | `pytest tests/test_turn_analytics_aggregator.py tests/test_release_gate.py::test_build_checks_includes_optional_replay_report tests/test_release_gate.py::test_non_blocking_check_is_warning`. | `logs/turn_analytics.jsonl` dataset + `logs/turn_analytics_summary.json` KPI summary derived from `logs/session.jsonl`. |
| **Canonical turn state**: each stage updates a typed per-turn state object, and that same structure is emitted to runtime logs for audit/review parity. | `PipelineState`, `CandidateHit`, `stage_rewrite_query(...)`, `stage_retrieve(...)`, `stage_rerank(...)`, `stage_answer(...)`, and `append_pipeline_snapshot(...)` in `src/testbot/sat_chatbot_memory_v2.py` + `src/testbot/pipeline_state.py`. | `BDD-MR-01` / `BDD-MR-02` / `BDD-AC-01` (all answer-path scenarios now transit a shared state object). | `behave features/memory_recall.feature features/answer_contract.feature`; state-shape sanity via code review/static typing in pipeline stages. | `pipeline_state_snapshot` rows in `logs/session.jsonl` (`stage`, `state`). |

## BDD scenario ID mapping used in this matrix

- `BDD-MR-01` → `Scenario: cited response when context is sufficient` (`features/memory_recall.feature`)
- `BDD-MR-02` → `Scenario: exact fallback when context is insufficient` (`features/memory_recall.feature`)
- `BDD-AC-01` → `Scenario: rejection of uncited factual response` (`features/answer_contract.feature`)
